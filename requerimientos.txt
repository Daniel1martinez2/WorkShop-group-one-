
Grupo: Daniel Martinez, Alejandro Valverde, Miguel Olivares


Contexto: 
El programa debe representar una escena de un cuento, permitiendo interacciones con él mouse o teclado con sus elementos principales, modificando características como la visibilidad, posición, tamaño, entre otros patrones de diseño;  brindando así una experiencia más inmeresiva para el lector con el cuento.

Requerimientos funcionales
-------------------------------
R1. El juego debe representar una escena de un cuento
R2. El Lector debe interactuar con al menos 5 elementos mencionados en el cuento. 
R3. El juego debe tener objetos interactuables
R4. Los objetos se deben crear a partir de un archivo txt
R5. El juego debe tener 4 pantallas
R6. El juego debe tener una pantalla de finalizado
R7. Las interacciones deben tener un hilo conductor
R8. El juego debe guardar un txt al finalizar 
R9. Desacuerdo a los elementos con los que se haya interactuado, se modificará el txt original visualmente las palabras correspondientes a los elementos.
R10. Las interacciones se darán por el mouse y teclado. 
--------------------------------
Requerimientos no funcionales
RNF1. Se debe usar el efecto Parallax
RNF2. Las interacciones deben ser evidentes para el   “lector”, mediante el uso de patrones de diseño. 

-------------------------------
Entidades: 
E1. Juego
E2. Objetos
